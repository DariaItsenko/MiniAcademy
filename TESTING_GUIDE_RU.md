## 🔍 Как проверить что все работает:

### Шаг 1: Откройте консоль разработчика

1. Запустите приложение: `npm run dev`
2. Откройте в браузере: `http://localhost:5173`
3. Нажмите **F12** (или правой кнопкой → "Inspect" → вкладка "Console")

### Шаг 2: Зарегистрируйте тестовый аккаунт

1. Нажмите **"Create Account"** (Создать аккаунт)
2. Введите:
   - **Username:** test123
   - **Password:** pass123
   - **Email:** test@example.com
   - **Age:** 7
   - **Grade:** 2

3. Нажмите **"Sign Up"**

### Шаг 3: Проверьте консоль

В консоли вы должны увидеть:

```
🎉 Creating new account!
Username: test123
Password: pass123
Email: test@example.com
Grade: 2
✅ Users saved successfully: ["test123"]
✅ Verification: Data persisted in localStorage
```

Если видите эти сообщения - **регистрация работает! ✅**

### Шаг 4: Проверьте localStorage вручную

В консоли разработчика введите:

```javascript
JSON.parse(localStorage.getItem('elementary_users'))
```

Вы должны увидеть ваш аккаунт:

```javascript
{
  test123: {
    password: "pass123",
    email: "test@example.com",
    grade: 2,
    points: 0,
    ...
  }
}
```

### Шаг 5: Выйдите из аккаунта

1. Нажмите кнопку **"Logout"** (Выход)
2. Вы вернетесь на экран входа

### Шаг 6: Войдите снова

1. Введите:
   - **Username:** test123
   - **Password:** pass123
2. Нажмите **"Login"**

В консоли вы должны увидеть:

```
🔐 Logging in user: test123 Grade: 2 Admin: false
✅ Login info saved to localStorage
📂 Loading user data from localStorage...
All users: ["test123"]
✅ User found: test123
User data: {password: "pass123", email: "test@example.com", ...}
💾 User data updated and saved
✅ Login complete!
```

Если видите эти сообщения - **вход работает! ✅**

## 🐛 Диагностика проблем

### Проблема 1: "Аккаунт не запоминается"

**Проверьте:**

1. В консоли введите:
```javascript
localStorage.getItem('elementary_users')
```

2. Если вернулось `null` - данные не сохраняются
3. Проверьте настройки браузера: разрешен ли localStorage?

**Решение:**
- Используйте обычный режим браузера (не инкогнито!)
- Проверьте: Settings → Privacy → Cookies разрешены

### Проблема 2: "User not found" при входе

**Проверьте:**

1. Нажмите "👥 Show Registered Accounts"
2. Убедитесь что ваш username есть в списке
3. Проверьте точно ли вводите username (без пробелов!)

**Решение:**
- Зарегистрируйтесь заново
- Или используйте кнопку "Use" из списка аккаунтов

### Проблема 3: localStorage очищается при обновлении

**Это НЕ нормально!**

**Проверьте:**
- Не используете ли режим инкогнито?
- Не установлены ли расширения для очистки куки?
- Не стоит ли автоочистка данных при закрытии браузера?

**Решение:**
- Отключите режим инкогнито
- Отключите автоочистку: Settings → Privacy → Clear browsing data → "Advanced" → Снимите галочку "Cookies and site data"

## 🔬 Расширенная диагностика

### Проверка 1: Сколько пользователей в базе?

В консоли:

```javascript
const users = JSON.parse(localStorage.getItem('elementary_users') || '{}');
console.log('Total users:', Object.keys(users).length);
console.log('Usernames:', Object.keys(users));
```

### Проверка 2: Посмотреть данные конкретного пользователя

```javascript
const users = JSON.parse(localStorage.getItem('elementary_users') || '{}');
console.log(users['test123']); // Замените test123 на ваш username
```

### Проверка 3: Кто сейчас залогинен?

```javascript
const currentUser = localStorage.getItem('elementary_current_user');
console.log('Current user:', currentUser);
```

### Проверка 4: Очистить все данные (ОСТОРОЖНО!)

**⚠️ Это удалит ВСЕ аккаунты!**

```javascript
localStorage.clear();
location.reload();
```
